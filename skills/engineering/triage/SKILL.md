# Triage

Mova issues no issue tracker do projeto por uma pequena máquina de estados de papéis de triagem.

Cada comentário ou issue postada no issue tracker durante a triagem **precisa** começar com este disclaimer:

```
> *This was generated by AI during triage.*
```

## Docs de referência

- [AGENT-BRIEF.md](AGENT-BRIEF.md) — como escrever agent briefs duráveis
- [OUT-OF-SCOPE.md](OUT-OF-SCOPE.md) — como a base de conhecimento `.out-of-scope/` funciona

## Papéis

Dois papéis de **categoria**:

- `bug` — algo está quebrado
- `enhancement` — nova feature ou melhoria

Cinco papéis de **estado**:

- `needs-triage` — mantenedor precisa avaliar
- `needs-info` — esperando o reporter para mais informação
- `ready-for-agent` — totalmente especificada, pronta para um agente AFK
- `ready-for-human` — precisa de implementação humana
- `wontfix` — não vai ser feito

Cada issue triada deve carregar exatamente um papel de categoria e um papel de estado. Se papéis de estado conflitam, sinalize e pergunte ao mantenedor antes de fazer qualquer coisa.

Estes são nomes de papéis canônicos — as strings de label de fato usadas no issue tracker podem diferir. O mapeamento deveria ter sido fornecido a você — rode `/setup-matt-pocock-skills` se não.

Transições de estado: uma issue sem labels normalmente vai para `needs-triage` primeiro; dali ela move para `needs-info`, `ready-for-agent`, `ready-for-human` ou `wontfix`. `needs-info` retorna para `needs-triage` uma vez que o reporter responde. O mantenedor pode sobrescrever a qualquer momento — sinalize transições que parecem incomuns e pergunte antes de prosseguir.

## Invocação

O mantenedor invoca `/triage` e descreve o que quer em linguagem natural. Interprete o pedido e aja. Exemplos:

- "Me mostre qualquer coisa que precise da minha atenção"
- "Vamos olhar a #42"
- "Mova a #42 para ready-for-agent"
- "O que está pronto para agentes pegarem?"

## Mostre o que precisa de atenção

Consulte o issue tracker e apresente três grupos, do mais antigo primeiro:

1. **Sem label** — nunca triadas.
2. **`needs-triage`** — avaliação em andamento.
3. **`needs-info` com atividade do reporter desde as últimas notas de triagem** — precisa de re-avaliação.

Mostre contagens e um resumo de uma linha por issue. Deixe o mantenedor escolher.

## Triagem de uma issue específica

1. **Reúna contexto.** Leia a issue completa (body, comentários, labels, reporter, datas). Faça o parse de quaisquer notas de triagem prévias para que você não re-pergunte questões já resolvidas. Explore a codebase usando o glossário de domínio do projeto, respeitando ADRs na área. Leia `.out-of-scope/*.md` e traga à tona qualquer rejeição prévia que se pareça com esta issue.

2. **Recomende.** Diga ao mantenedor sua recomendação de categoria e estado com raciocínio, mais um resumo breve da codebase relevante à issue. Espere direção.

3. **Reproduza (apenas bugs).** Antes de qualquer grilling, tente reprodução: leia os passos do reporter, trace o código relevante, rode testes ou comandos. Reporte o que aconteceu — repro bem-sucedido com code path, repro falho ou detalhe insuficiente (um sinal forte de `needs-info`). Um repro confirmado faz um agent brief muito mais forte.

4. **Faça grilling (se necessário).** Se a issue precisa ser desenvolvida, rode uma sessão de `/grill-with-docs`.

5. **Aplique o resultado:**
   - `ready-for-agent` — poste um comentário de agent brief ([AGENT-BRIEF.md](AGENT-BRIEF.md)).
   - `ready-for-human` — mesma estrutura de um agent brief, mas anote por que não pode ser delegada (julgamentos, acesso externo, decisões de design, teste manual).
   - `needs-info` — poste notas de triagem (template abaixo).
   - `wontfix` (bug) — explicação polida, depois feche.
   - `wontfix` (enhancement) — escreva em `.out-of-scope/`, linke a partir de um comentário, depois feche ([OUT-OF-SCOPE.md](OUT-OF-SCOPE.md)).
   - `needs-triage` — aplique o papel. Comentário opcional se houver progresso parcial.

## Sobrescrita rápida de estado

Se o mantenedor diz "mova a #42 para ready-for-agent", confie nele e aplique o papel direto. Confirme o que você está prestes a fazer (mudanças de papel, comentário, fechar), depois aja. Pule grilling. Se mover para `ready-for-agent` sem uma sessão de grilling, pergunte se ele quer escrever um agent brief.

## Template de needs-info

```markdown
## Triage Notes

**What we've established so far:**

- ponto 1
- ponto 2

**What we still need from you (@reporter):**

- pergunta 1
- pergunta 2
```

Capture tudo resolvido durante o grilling sob "established so far" para que o trabalho não seja perdido. Perguntas precisam ser específicas e acionáveis, não "por favor forneça mais info".

## Retomando uma sessão anterior

Se notas de triagem prévias existem na issue, leia-as, cheque se o reporter respondeu alguma questão pendente e apresente um quadro atualizado antes de continuar. Não re-pergunte questões já resolvidas.
